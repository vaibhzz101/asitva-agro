🚀 EmailJS Client-Side Setup (Fixed)

⚠️ Issue Fixed!

Problem: EmailJS doesn't work from server-side (Next.js API routes)

Solution: Moved EmailJS to client-side (browser) - this is the correct way!

📧 Step 1: Create EmailJS Account

  1. Go to EmailJS.com
  2. Click "Sign Up"
  3. Create account with any email
  4. Verify your email address

⚙️ Step 2: Add Email Service

  1. Go to "Email Services" in dashboard
  2. Click "Add New Service"
  3. Choose "Gmail"
  4. Service ID: service_astitva
  5. Connect your Gmail account (any Gmail works)
  6. Click "Create Service"

📝 Step 3: Create Email Template

  1. Go to "Email Templates"
  2. Click "Create New Template"
  3. Template ID: template_contact
  4. Copy this template content:
Subject: 🌾 New Website Inquiry from {{from_name}} - Astitva Agro Dear Astitva Agro Team, You have received a new contact form submission from your website: 📋 CONTACT DETAILS: 👤 Name: {{from_name}} 📧 Email: {{from_email}} 📱 Phone: {{phone}} 🏢 Company: {{company}} 🌾 Product Interest: {{product_interest}} 💬 MESSAGE: {{message}} 📅 Submitted: {{timestamp}} 🌐 Source: {{website_source}} --- IMPORTANT: Please respond directly to the customer's email: {{reply_to}} This email was automatically sent from your website contact form. Best regards, Astitva Agro Website System

Template Settings:

  • To Email: info@astitvaagrofpcl.com
  • From Name: Astitva Agro Website
  • Reply To: {{reply_to}}

🔑 Step 4: Get Your Public Key

  1. Go to "Account" → "General"
  2. Copy your Public Key
  3. It should look like: user_xxxxxxxxxxxxxxxxx

🔧 Step 5: Update Your Code

Replace YOUR_PUBLIC_KEY in the Contact component:

// In components/Contact.tsx, line ~85 ;(window as any).emailjs.init("YOUR_ACTUAL_PUBLIC_KEY_HERE")

Replace with your actual public key from EmailJS dashboard.

✅ How It Works Now (Client-Side)

  1. EmailJS script loads in the browser
  2. User fills form and clicks "Send Message"
  3. EmailJS sends email directly from browser
  4. Email delivered to info@astitvaagrofpcl.com
  5. User sees success message

✅ Benefits of Client-Side:

  • Works with static hosting (Hostinger)
  • No server configuration needed
  • EmailJS designed for this approach
  • More reliable delivery

🛠️ Testing Steps

  1. Update your public key in the code
  2. Build and deploy your website
  3. Open the contact form
  4. Wait for "Email service ready" green message
  5. Fill out the form and submit
  6. Check info@astitvaagrofpcl.com for the email

🔍 Troubleshooting

  • "Email service not loaded"? → Check internet connection
  • "Failed to send email"? → Verify public key is correct
  • Wrong service/template IDs? → Check they match exactly
  • Emails not arriving? → Check spam folder first
  • Console errors? → Open browser dev tools to see errors
